# Change Proposal

Describe what is changing and why.

## Instructions

1. Read the current-state analysis to understand what exists today
2. If a JIRA epic is referenced, fetch it and its stories via MCP
3. Summarize:
   - **Problem**: What need or gap is driving this change?
   - **Scope**: Which crates/services are affected?
   - **Approach**: High-level strategy (add features, remove deprecated
     code, restructure domain model, or combination)
4. List each capability being added, modified, or removed with a
   one-line summary and source (JIRA story, design decision, tech debt)
5. Call out risks: cross-service contract changes, breaking changes,
   complexity concerns

## Output

A concise proposal document. This is reviewed by a human before
specs and design are created.
